# Contributing to RunX

Thank you for your interest in contributing to RunX! We welcome community contributions to help make RunX the ultimate elite fitness platform.

## Code of Conduct

By participating in this project, you agree to abide by our code of conduct:
*   Be respectful and professional.
*   Focus on technical excellence.
*   Maintain the monochrome design language.

## How to Contribute

1.  **Search for Issues:** Check the issue tracker before starting work.
2.  **Fork and Branch:** Create a feature branch for your changes.
3.  **Code Standards:** Follow the existing architecture and the monochrome design guidelines.
4.  **Test Your Changes:** Ensure all tests pass (`flutter test`).
5.  **Submit a PR:** Provide a detailed description of your changes.

## Architecture Guidelines

RunX follows a service-oriented architecture:
*   **Models:** Data structures in `lib/models`.
*   **Services:** Business logic and data fetching in `lib/services`.
*   **Features:** UI components and stateful logic in `lib/features`.
*   **Widgets:** Reusable UI components in `lib/widgets`.

## UI Guidelines

RunX uses a strict monochrome palette.
*   Primary: #FFC107 (Amber for highlights only)
*   Background: #121212
*   Surface: #1E1E1E
*   Typography: Bold, high-contrast, letter-spaced.

We look forward to your contributions!
